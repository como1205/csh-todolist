# CSH-TodoList 도메인 정의서

## 1. 도메인 개요

**csh-todolist**는 사용자 인증 기반의 개인 할일 관리 애플리케이션입니다. 사용자는 개인의 할일을 생성, 조회, 수정, 삭제할 수 있으며, 국경일과 같은 공통 일정도 함께 관리할 수 있습니다.

## 2. 핵심 도메인 모델

### 2.1 User (사용자)
- **목적**: 애플리케이션 사용자를 식별하고 인증
- **주요 속성**:
  - 사용자 ID
  - 이메일
  - 비밀번호
  - 가입일시
- **주요 행위**:
  - 회원가입
  - 로그인/로그아웃

### 2.2 TodoItem (할일)
- **목적**: 사용자가 관리하는 개별 할일
- **주요 속성**:
  - 할일 ID
  - 소유자 (User 참조)
  - 제목
  - 내용
  - 시작일시
  - 만료일시
  - 상태 (진행중, 완료, 삭제됨)
  - 생성일시
  - 수정일시
- **주요 행위**:
  - 생성
  - 조회
  - 수정
  - 삭제 (휴지통으로 이동)
  - 복원 (휴지통에서 복구)
  - 영구 삭제

### 2.3 PublicHoliday (공휴일/국경일)
- **목적**: 모든 사용자에게 공통으로 표시되는 일정
- **주요 속성**:
  - 공휴일 ID
  - 이름
  - 날짜
  - 설명
  - 반복 여부
- **주요 행위**:
  - 조회 (읽기 전용)
  - 관리자만 생성/수정 가능
  - 삭제 불가

### 2.4 RecycleBin (휴지통)
- **목적**: 삭제된 할일을 임시 보관
- **주요 속성**:
  - 삭제된 TodoItem 참조
  - 삭제일시
- **주요 행위**:
  - 삭제된 항목 조회
  - 원복
  - 영구 삭제

## 3. 주요 유스케이스

### 3.1 인증 및 권한
- 회원가입
- 로그인
- 로그아웃
- 인증된 사용자만 할일 관리 기능 접근 가능

### 3.2 할일 관리
- **생성**: 새로운 할일 추가 (제목, 내용, 시작일, 만료일)
- **조회**: 본인의 할일 목록 조회 (공휴일 포함)
- **수정**: 기존 할일의 일정 및 내용 수정
- **삭제**: 할일을 휴지통으로 이동
- **복원**: 휴지통의 할일을 원복
- **영구삭제**: 휴지통의 할일을 완전 삭제

### 3.3 일정 표시
- 개인 할일과 공휴일을 통합하여 표시
- 만료 기간 기반 할일 정렬 및 필터링

## 4. 비즈니스 규칙

### 4.1 인증 규칙
- 모든 주요 기능은 로그인 후 사용 가능
- 사용자는 본인의 할일만 조회/수정/삭제 가능

### 4.2 할일 관리 규칙
- 할일은 시작일시와 만료일시를 가짐
- 삭제된 할일은 즉시 제거되지 않고 휴지통으로 이동
- 휴지통의 할일은 복원 또는 영구 삭제 가능

### 4.3 공휴일 규칙
- 공휴일은 모든 사용자에게 동일하게 표시
- 일반 사용자는 공휴일을 삭제할 수 없음
- 공휴일은 사용자의 개인 할일과 구분되어 관리

## 5. 도메인 용어 정의

| 용어 | 정의 |
|------|------|
| TodoItem | 사용자가 생성한 개인 할일 항목 |
| PublicHoliday | 국경일, 공휴일 등 공통 일정 |
| RecycleBin | 삭제된 할일이 임시 보관되는 공간 |
| 만료일시 | 할일이 완료되어야 하는 목표 시점 |
| 원복 | 휴지통의 할일을 다시 활성 상태로 복구 |
| 영구삭제 | 휴지통의 할일을 되돌릴 수 없게 완전 제거 |

## 6. 도메인 관계도

```
User (1) ──< (N) TodoItem
                    │
                    │ (삭제시)
                    ▼
              RecycleBin (1:1)

PublicHoliday (독립적으로 모든 사용자에게 표시)
```

## 7. 제약사항 및 고려사항

- 사용자별 데이터 격리 보장 필요
- 공휴일 데이터는 중앙에서 관리
- 휴지통 보관 기간 정책 고려 가능 (추후 확장)
- 만료일 기반 알림 기능 고려 가능 (추후 확장)
